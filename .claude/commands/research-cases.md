---

**아래 [리서치 산출 설정]과 [상세검색 설정]을 먼저 확정해 `검색설정 블록`을 만든 뒤, 그 블록을 실행 프롬프트에 포함하여 `case-researcher` 서브에이전트를 실행하라.** (순서: 설정 확정 → 블록 생성 → 블록 실은 Task 호출)

- 검색 키워드 / 사실관계 / 검색설정: $ARGUMENTS  (구분자 `|` — 1번째=키워드, 2번째=사실관계, 3번째(선택)=검색설정 인라인. 사실관계 없으면 사용자에게 1~3문장 요청)
- 사이트는 lbox. **도메인**(`https://lbox.kr`)을 쓴다. 시작 전 **로그인 상태를 먼저 확인**하고, 안 돼 있으면 멈추고 사용자에게 직접 로그인을 요청할 것(자격증명 입력·저장·추측 금지).

- **[리서치 산출 설정 확정 — 시작 시 반드시 확인]** 키워드·사실관계 외에, 최종 산출물(docx)을 좌우하는 아래 2가지를 **인라인에 없으면 `AskUserQuestion`으로 물어** 확정한다:
  1. **최종본 상위 N** — 최종 docx에 담을 판례 수(기본 **30**, 예 20/30/50/기타). 나머지 부합건은 백업으로만 기록.
  2. **유용성 방향성(골드기준)** — 최종본에서 판례의 유용성을 판정하는 기준 방향. 즉 "어떤 논증을 뒷받침하는 판례가 가장 유용한가". 예: *"피고인이 불법촬영(카촬) 등 다른 불리한 정황이 있음에도, 피해자의 거부 발언·일부 응하는 정황 등을 근거로 준강간은 별도로 심신상실·항거불능을 인정하지 않아 무죄로 본 판례"*. 이 방향성이 Phase3 정밀 재평가·정렬(유용성순)과 bold 대상 선정의 기준이 된다.
  - 확정한 `상위N`·`유용성방향성` 을 검색설정 블록·서브에이전트 프롬프트에 함께 전달한다.

- **[상세검색 설정 확정 — 하이브리드, 서브에이전트 실행 전 메인 세션이 수행]** 리서치를 시작할 때마다 사용자가 lbox 상세검색 설정을 직접 고르게 한다. 선택지·값의 정본(SSOT)은 `case-researcher` 에이전트 정의의 **'상세검색 설정 카탈로그'** 절이다.
  - **(a) 인라인이 있으면**($ARGUMENTS 3번째 구획의 `키=값` 토큰) 그대로 파싱해 쓰고, 빠진 항목만 완전성 기본값으로 채운다. 지정 예:
    - 완전성: `사건유형=형사 법원=전체 정렬=관련도순 재판유형=전체 선고일=전체 주문유형= 탭=판례 조건검색="완전일치:고가","제외:코인","법조문:형법347" 모드=완전성`
    - 빠른탐색: `모드=빠른탐색 페이지상한=3 정렬=관련도순 사건유형=형사`
  - **(b) 인라인이 없으면 `AskUserQuestion`으로 클릭 선택**을 받는다. **AskUserQuestion은 질문당 옵션 ≤4개**이므로 5개 이상인 축은 "기타(직접입력)"로 접거나 질문을 분할한다(사용자는 항상 '기타'로 자유값 입력 가능). 핵심 차원(각 질문 기본=권장 옵션):
    - ① 모드: 완전성(권장)/빠른탐색  — 빠른탐색이면 페이지상한 N을 후속 질문(3/5/10/기타)으로 받아 `페이지상한=N`에 반영.
    - ② 사건유형: 형사(권장)/전체/민사/기타(자유입력)  — 형사=이 도구의 형사리서치 기본, 비형사면 전체/해당유형(에이전트 카탈로그 2번 참조).
    - ③ 정렬: 관련도순(권장)/최신순/인용순/기타(입수일·조회 등 자유입력).
    - ④ 재판유형: 전체(권장)/판결/결정.
    - ⑤ 선고일: 전체(권장)/최근3년/최근10년/기타(자유입력: 최근1·5년 또는 YYYYMMDD-YYYYMMDD).
    - ⑥ 법원: 전체(권장)/특정(선택 시 법원명 자유입력).
    - ⑦ 조건검색·주문유형: 필요하면 `필드:값`(조건검색)·주문유형 값을 자유입력으로 추가(기본 없음/비움).
  - **완전성 프리셋(기본값)**: 사건유형=형사(형사리서치 최적화 기본·매번 확인, 비형사면 전체)·법원=전체(모든 심급)·**정렬=관련도순**·주문유형=비움·재판유형=전체·선고일=전체·탭=판례·조건검색=없음·페이지상한=없음(끝까지). **빠른탐색 프리셋**: 정렬=관련도순·**`페이지상한=N`(기본 3)**·법원=전체·나머지 완전성과 동일 — "상위 N페이지만 → 누락 0 보장 불가"를 명시.
  - 확정 결과를 **`검색설정 블록`(위 키=값 정규형)으로 정리해 서브에이전트 실행 프롬프트에 함께 전달**한다. 서브에이전트는 이를 그대로 적용·기록하고 사용자에게 되묻지 않는다.
- **실행 전 권한 확인**: worklist는 `Write(data/**)`로 통째 재기록한다(Edit 미사용 — 다른 환경에서 권한 프롬프트로 흐름이 끊기지 않게).
- **무중단 연속 실행(기본)**: 배치 사이·Phase1↔Phase2↔Phase3 사이에 **멈추지 않고 사용자 지정 상한까지 한 번에** 진행한다(승인 대기 게이트 없음). 컨텍스트 관리를 위한 자체 배치 분할·fresh 재개는 허용(그건 멈춤이 아님). 멈추는 건 ⓐ상한 도달 ⓑ차단 징후 ⓒ로그인 필요, 또는 사용자가 "테스트로 Phase1까지만" 등 **명시적으로 중간 정지를 지시**한 경우뿐.
- **속도(배치1 지연 교정)**: 필터·정렬·페이지 이동 등 반복·재렌더 요소는 **`role=`/텍스트 셀렉터로 바로 클릭**(고정 ref 재사용·ref 찾기용 재스냅샷·클릭 재시도 루프 금지). 페이지당 스냅샷 1회. 필터 적용 확인은 총건수 텍스트만으로.
- **★페이지네이션(치명버그 교정, 필수)**: lbox 번호형은 『숫자 5개 창+화살표』이고 **`다음으로 이동`은 1페이지가 아니라 +5(그룹) 점프**다(실측). **단일 페이지는 숫자 버튼(`role=button[name="57"]`)으로만** 이동하고, '다음으로 이동'은 다음 5개 창을 드러낼 때만 사용한다. 창 전환마다 보이는 숫자 창 첫 값이 기대값과 같은지 검증(어긋나면 중단·보고). 상세는 case-researcher 정의 §1-2 [A]. 페이지를 건너뛰면 인벤토리가 표본이 되어 완전성이 깨진다.
- **Phase1(목록 트리아지)**: 전달한 `검색설정 블록`을 적용(완전성 기본=법원 전체·모든 심급, 주문유형 비움, 정렬 관련도순)해 (상한까지) 캡처하고 `data/cases/{슬러그}/_worklist.md` 인벤토리를 완성. 죄목태그 1차 필터는 false-positive 우호(애매·태그무는 통과·보류; **단 사용자 특수필터가 있으면 그 지시 우선**). **정렬=관련도순은 비결정적이므로 완전성 검증(행수=총건수, 모든 심급)을 엄격 적용.** 총건수가 화면에 없거나 빠른탐색(상위 N)이면 "누락 0 보장 불가"를 worklist·최종보고에 명시. 검증 후 **곧바로 Phase2로 이어감**.
- **Phase2(전문 수집·분석·저장)**: 각 통과·보류 판례를 **`/case/{법원}/{사건번호}` 직접링크로 열어** `browser_evaluate(()=>main.innerText, filename="data/cases/{슬러그}/raw/{사건번호}.md")`로 **원문을 파일에 덤프**(모델 토큰 0 — 전문을 컨텍스트로 읽지 말 것). 대량(수백 건)이면 **수집(직렬, 단일 브라우저)과 오프라인 분석(병렬 워크플로)을 겹쳐** 돌린다(case-researcher §Phase3-0 파이프라인; 메인 세션이 수집 에이전트 ∥ 분석 워크플로를 동시에 오케스트레이션. **서브에이전트는 다른 에이전트를 스폰하지 않음**). 분석은 각 원문을 정독해 사실관계 유사성·부합 발췌·확정여부·심급을 판정·파일 저장. 확정여부·동일사건 심급은 `/case/` 페이지의 **'상•하위 판결 N [확정라벨]'** 영역에서 확인. (소수면 결과목록 카드클릭→인페이지 패널 방식도 가능.) 끝나면 Phase3로 이어감.
- **Phase3(최종 산출물 docx)**: 부합 판례 중 **유용성 상위 N건**만(§리서치 산출 설정) `.claude/style_case_research.docx` 템플릿 표로 만든다. **유용성 정렬 = ①유용성 방향성 부합도(최우선·최종 기준) → ②확정 여부 → ③심급(지방<고등<대법)**. **판례번호 셀 = 순수 형식 + lbox `/case/` 하이퍼링크**(클릭 열람), **내용 칸 = 【사실관계】+【법원의 특징적 설시】에 핵심만 bold**(전형적 '증명부족/무죄' 문구 제외). 산출: `output/{슬러그}_판례리서치_최종.docx`(상위 N) + 전체 부합 백업 docx + `_final_ranking.json`(전체 랭킹). 상세 규약·스크립트는 case-researcher §Phase3 및 `scripts/` 참조.
- 페이싱·차단회피·로그인·실검증의 정본은 `docs/lbox-pacing-and-verification.md` 준수: 클릭 위주, 직접 fetch 금지, 목록 복귀는 **상세 패널 닫기(닫기/Escape) 우선**(전문이 인페이지 패널이라 페이지 이동 없이 복귀), 동작 사이 가변 텀, 차단 징후 시 worklist Write 후 중단·보고. **저장/worklist 포맷의 정본은 case-researcher 에이전트 정의**(충돌 시 에이전트 본문 우선).
- 끝에 총건수(또는 관찰 행수)·심급 분포·통과/보류/제외 분포·저장 사건번호 목록·완전성 보장 여부·미해결 〔확인 필요〕·재개 지점을 보고할 것.
