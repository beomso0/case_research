# lbox 판례 리서치 도구 (Claude Code)

변호사 실무를 위한 **판례 대량 수집 → 병렬 분석 → 유용성 순위 정리(docx)** 자동화 도구입니다. lbox.kr(로그인 필요·유료 판례 DB)에서 특정 쟁점의 판례를 **하나도 빠뜨리지 않고 모아**, 세우신 논증에 **유용한 순서로** 정리한 표(Word)로 만들어 드립니다.

핵심 슬래시 커맨드는 하나, **`/research-cases`** 입니다. (구동 엔진은 `case-researcher` 서브에이전트입니다.)

---

## ⚡ 설치 (Windows / macOS)

**사전 요구**: **Git**, **Node.js**(`npx`), **Python 3**(`pip`), 그리고 **Claude Code**.
- Node.js: [nodejs.org](https://nodejs.org) — 또는 `winget install OpenJS.NodeJS`(Windows) / `brew install node`(macOS)
- Python 3: [python.org](https://www.python.org) — 또는 `winget install Python.Python.3`(Windows) / `brew install python`(macOS)

**Windows (PowerShell)**
```powershell
git clone https://github.com/beomso0/claude_test.git
cd claude_test
pip install -r requirements.txt
```

**macOS (터미널)**
```bash
git clone https://github.com/beomso0/claude_test.git
cd claude_test
pip3 install -r requirements.txt
```

설치 후 이 폴더를 **Claude Code로 열고** 아래 [Quick Start](#quick-start)를 따라 주세요. 브라우저(Playwright) MCP는 최초 실행 시 `npx`가 자동으로 내려받으므로 별도 설치가 필요 없습니다. 자세한 셋업은 [docs/setup.md](docs/setup.md)를 참고해 주세요.

## 왜 유용한가요
- **누락 없는 수집**: 검색결과를 끝까지·모든 심급(하급심 포함) 트리아지하고, "조금이라도 관련되면 포함"(false-positive 우호)해 놓친 판례를 최소화합니다. 최종 취사선택은 변호사가 하십니다.
- **빠릅니다**: 전문을 `/case/` 직접링크로 열어 파일에 바로 덤프하고(모델이 전문을 토큰으로 다시 읽지 않습니다), **수집(직렬)과 분석(병렬)을 겹쳐** 돌려 수백 건도 현실적인 시간에 처리합니다.
- **논증 중심 정리**: 단순 요약이 아니라, **지정하신 '유용성 방향성'(어떤 논증을 뒷받침하는 판례가 유용한가)** 기준으로 재평가·정렬하고, 표의 각 판례에 **핵심 사실관계와 특징적 설시를 bold**로 짚어 드립니다. **판례번호에는 lbox 원본 하이퍼링크**가 걸려 바로 열람하실 수 있습니다.
- **인용 정확성**: 사건번호·법조문·판시는 화면 문자열만 기록하고 지어내지 않습니다. 원문은 전부 보존됩니다(감사·재검토용).

## 핵심 프로세스 (3단계)
```
Phase 1  목록 트리아지     검색결과를 끝까지 캡처 → 부합 후보 인벤토리(_worklist.md)
   │
Phase 2  전문 수집·분석     각 판례 /case/ 직접링크 → 원문 파일 덤프(raw/)         ← 수집(직렬)
   │                        저장된 원문을 여러 에이전트가 나눠 정독·유사성 판정     ← 분석(병렬, 동시)
   │
Phase 3  최종 산출물(docx)  부합건 중 '유용성 방향성'으로 정밀 재평가 → 상위 N 표
                            (판례번호 하이퍼링크 + 사실관계·설시 bold, 유용성순 정렬)
```
**완전성**(누락 0)과 **속도**를 함께 추구합니다. 자세한 구동 규칙은 [.claude/agents/case-researcher.md](.claude/agents/case-researcher.md), 파이프라인 스크립트는 [scripts/](scripts/)에 있습니다.

## Quick Start
1. **설치**: 위 [설치](#-설치-windows--macos) 절을 따라 주세요(클론 + `pip install -r requirements.txt`).
2. **로그인**: 세션에서 브라우저가 열리면 **직접** lbox.kr에 로그인해 주세요(도구는 자격증명을 입력·저장하지 않습니다).
3. **실행**: 아래 한 줄이면 시작 시 필요한 값을 순서대로 여쭤봅니다.
   ```
   /research-cases 준강간
   ```
   또는 한 번에 모두 지정하실 수도 있습니다.
   ```
   /research-cases 준강간 | 술 취해 자던 여성 상대 성관계, 피해자가 거부 말·일부 응한 정황 → 항거불능 부정 무죄 | 사건유형=형사 주문유형=무죄 재판유형=판결 선고일=최근10년 | 상위N=30 유용성방향성="카촬 등 다른 불리정황에도 준강간은 심신상실 인정 않아 무죄"
   ```
4. **결과**: `output/<슬러그>_판례리서치_최종.docx`(상위 N, 유용성순)와 전체 백업본, 원문·분석 데이터가 생성됩니다.

## `/research-cases` — 시작 시 입력값 상세

입력은 `|` 로 구분합니다: **`키워드 | 사실관계 | 검색설정 | 산출설정`**. 인라인에서 빠뜨린 값은 **시작 시 클릭형 질문(AskUserQuestion)으로 여쭤** 확정합니다(키워드만 주셔도 됩니다).

### 1) 키워드 (필수)
lbox 검색어를 넣어 주세요(예 `준강간`). 정밀 조정은 아래 `조건검색`으로 하실 수 있습니다.

### 2) 사실관계 (권장 — 없으면 여쭤봄)
찾으시는 판례의 사실관계·쟁점을 1~3문장으로 적어 주세요. Phase2에서 각 판례가 이 사실관계에 부합하는지 판단하는 기준이 됩니다.

### 3) 검색설정 (lbox 상세검색 — 미지정 시 기본값)
| 키 | 값 | 기본값 |
|---|---|---|
| `사건유형` | 형사·민사·행정·가사·특허·기타 (쉼표 다중) | **형사** (형사리서치 최적화; 비형사면 지정) |
| `법원` | 전체 / 특정법원명(쉼표) | **전체**(모든 심급) |
| `정렬` | 관련도순·최신순·입수일순·조회순·인용순 | **관련도순** |
| `재판유형` | 전체·판결·결정 | **전체** |
| `선고일` | 전체·최근1/3/5/10년·`YYYYMMDD-YYYYMMDD` | **전체** |
| `주문유형` | 비움 / 무죄·징역·벌금… | **비움**(무·유죄 모두) |
| `조건검색` | `"필드:값"` (완전일치·제외·사건명·주문·법조문·당사자명·판결요지·사건부호·청구취지·신청취지) | 없음 |
| `모드` / `페이지상한` | 완전성 / 빠른탐색·상위 N페이지 | **완전성**(끝까지) |

### 4) 산출설정 (최종 docx — 시작 시 반드시 확인) ★이 도구의 핵심
| 항목 | 의미 | 기본값 |
|---|---|---|
| `상위N` | 최종본에 담을 판례 수(나머지는 백업 기록) | **30** |
| `유용성방향성` | **최종본에서 유용성을 판정하는 기준 방향** = "어떤 논증을 뒷받침하는 판례가 가장 유용한가". Phase3 재평가·정렬·bold의 기준입니다. | (없으면 여쭤봄) |

> **유용성 방향성 예시**: *"피고인이 불법촬영(카촬) 등 다른 불리한 정황이 있음에도, 피해자의 거부 발언·일부 응하는 정황을 근거로 준강간은 별도로 심신상실·항거불능을 인정하지 않아 무죄로 본 판례"*. 이 방향성이 구체적일수록 최종본이 정확히 논증에 맞춰 정렬됩니다.

### 입력 방식 (둘 다 가능합니다)
- **대화형**: `/research-cases 준강간` 처럼 최소만 주시면, 사실관계·검색설정·상위N·유용성방향성을 **클릭형 질문**으로 하나씩 확정합니다.
- **인라인**: 위 `키=값`을 `|` 구획에 한 줄로 넣어 주세요. 지정하지 않은 항목만 기본값이 적용됩니다.

## 최종 산출물
- **`output/<슬러그>_판례리서치_최종.docx`** — 상위 N건, **유용성 내림차순**(①방향성 부합도 → ②확정 여부 → ③심급 지<고<대)으로 정렬됩니다. 표 컬럼: **판례번호(lbox 하이퍼링크)** · 유무죄 · 확정여부 · 내용(【사실관계】+【법원의 특징적 설시】, 핵심 bold) · 비고(활용 방향).
- `output/<슬러그>_판례리서치.docx` — 전체 부합건 백업본입니다.
- `data/cases/<슬러그>/` — `raw/`(원문) · `analysis/`·`refined/`(분석) · `_worklist.md`(인벤토리) · `_final_ranking.json`(전체 랭킹).

## 주의사항 / 한계
- **로그인은 사람만 합니다.** 도구는 자격증명을 입력·저장·추측하지 않습니다. 본인인증 팝업이 뜨면 멈추고 알려 드립니다.
- **약관 준수·본인 계정 범위**에서만 수집합니다. 요청 간격을 두고, 차단 징후(캡차/403/본인인증) 시 즉시 중단·보고합니다.
- **완전성 한계는 명시해 드립니다.** `빠른탐색`(상위 N페이지)이나 정렬=관련도순으로 상위 일부만 볼 때는 "누락 0 보장 불가"를 보고합니다. 완전 전수는 `완전성` 모드 + 결정적 정렬(최신순 등)로 하실 수 있습니다.
- **인용 정확성**을 지킵니다. 표는 원문 발췌에서만 만들고 지어내지 않습니다. 생성물은 **초안**이므로 변호사의 검토·책임 하에 사용해 주세요.
- **개인정보 보호**: 수집 자료·산출물(`data/`, `output/`)과 로그인 프로필(`.browser-profile/`)은 git에 커밋되지 않습니다.

자세한 동작 원리는 [CLAUDE.md](CLAUDE.md), 셋업은 [docs/setup.md](docs/setup.md), 수집 페이싱·차단회피 정본은 [docs/lbox-pacing-and-verification.md](docs/lbox-pacing-and-verification.md)를 참고해 주세요.
